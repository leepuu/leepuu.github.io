# 🔁 :where(…)

- `:where(…)` 가상 클래스 함수는 **부모 태그는 다른데 자식은 같은 태그**에 공통적인 스타일을 부여할 때 사용한다.
- 기존에는 부모 태그 + 중복 자식 태그를 나열하여 작성하는데 `:where()`가상 선택자를 사용하면 **부모를 한 그룹** 으로 만들어 간단하게 작성할 수 있다.

```scss
// Before
.header p,
.footer p,
.container p {
  background: red;
}

// After
:where(.header, .footer, .container) p {
  background: red;
}
```

<hr>

# 🔁 :is(…)

- `:is(…)` 가상 클래스 함수도 `:where(…)` 처럼 여러 요소의 스타일 지정 가능

```scss
// Before
.header p,
.footer p,
.container p {
  background: red;
}

// After
:is(.header, .footer, .container) p {
  background: red;
}
```

# 🔁 :where과 :is의 차이

- 설명을 보면 `:where`과 `:is`가 같은 의미로 사용이 되는데, 다르게 쓰는 이유는???

  - 명시도가 다르기 때문에!!!

  <div class="alert">
    <h6>명시도</h6>
    <p>요소에 적용하는 스타일 반영 순위로 흔히 우리가 아는 적용되는 우선 순위이다.</p>
  </div>

- `where`의 명시도는 0 / `is`는 `where`보다 높다

```scss
:is(.header, .container, .footer) p {
  background: salmon; // 해당 스타일이 적용
}

:where(.header, .container, .footer) p {
  background: skyblue;
}
```
