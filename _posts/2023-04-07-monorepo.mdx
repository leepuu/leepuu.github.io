# 🗂️ 모노레포란?

> **_하나의 레포지토리에서 서로 다른 여러 프로젝트들을 관리_**하는 소프트웨어 개발 전략

<br>

# 🗂️ 멀티레포 vs 모노레포

## 👩‍👦‍👦멀티레포

각 프로젝트가 독립적인 하나의 레포지토리를 만들어 관리, 이에 따라 **_여러 개의 레포지토리_**가 생성

| 장점 👍                                                                    | 단점 👎                                               |
| -------------------------------------------------------------------------- | ----------------------------------------------------- |
| 각 프로젝트 별 자율성이 높음                                               | 프로젝트를 생성할 때 마다 개발 환경, 배포 환경을 구성 |
| 각 프로젝트 별 독립성이 높아 서로 충돌 가능성이 적음                       | 코드 재사용이 쉽지 않아, 코드 중복되기 쉬움           |
| 레포별 크기가 작기 때문에 레포 훅을 기반으로 동작하는 도구들의 속도가 빠름 | 코드 리뷰가 나뉘어짐                                  |
|                                                                            | 프로젝트가 많아질수록 관리가 어려움                   |

## 🤷모노레포

각 프로젝트를 모두 **_하나의 레포지토리에서 일괄 관리_**
|장점 👍 | 단점 👎|
|---|---|
|모든 서비스가 연동된 상태에서 손쉽게 하나의 버전으로 관리 가능|레포의 크기가 크기 때문에 레포 훅을 기반으로 동작하는 도구들의 속도가 느림|
|코드의 공유와 재사용이 용이|레포의 거대화|
|의존성 관리가 쉬움<br>전체 서비스의 의존 관계가 한 리포지토리에서 확인 및 설정 가능||
|여러 프로젝트팀 간의 협업이 쉬움||

<div class="alert">
  <h6>그럼 모노레포가 무조건 좋을까?</h6>
  <p>🙅‍♀️그렇지 않다. 멀티레포의 장점이 모노레포의 단점이고 서로의 장단점이 교차하기 때문에 적절한 상황에 사용해야 함<br>모노레포의 적절한 상황은 아래와 같음</p>

    <li>호스트 애플리케이션을 플러그인 등으로 확장할 때</li>
    <li>공통 기능을 재사용하는 관련된 프로젝트의 집합</li>

# 🗂️ 모노레포 구현 도구

모노레포를 구축하기 위한 도구에는 [2022.stateofjs.com](https://2022.stateofjs.com/en-US/) 기준으로 Yarn Workspaces, npm Workspaces, Lerna, pnpm, Nx 등 이 있다.
![모노레포](/assets/blog/posts/github/monorepo/img1.png)

<br>

## Lerna

[Lerna](https://lerna.js.org/)는 가장 유명하고 많은 레퍼런스가 있는 도구로, Babel 개발자에 의해 개발되어 2015년부터 모노레포 관리 도구로 사용  
2022년 4월 개발자의 번 아웃으로 프로젝트 유지를 하지 않겠다 하였는데, 2022년 5월 현재 Nx의 개발사인 Nrwl이 프로젝트 관리 권한을 인수  
2021년까지는 제일 많이 사용되어왔지만, 2022년에는 3위로 하락

<br>

## Yarn Workspace, Npm Workspace

[Yarn(1.x)](https://classic.yarnpkg.com/lang/en/docs/workspaces/) 또는 [npm(7.x)](https://docs.npmjs.com/cli/v7/using-npm/workspaces)의 workspaces 필드를 사용해 간단히 모노레포를 구성

<br>

## Nx

구글 개발자들이 만든 오픈소스 프로젝트로 모노레포 구성을 위한 다양한 개발 도구를 제공하고 Angular, React와 같은 **_프론트엔드 프레임워크 기반의 개발 환경 구성_**뿐 아니라 Express, Nest.js와 같은 백엔드 기술 기반의 개발까지 폭넓게 지원

<br>
